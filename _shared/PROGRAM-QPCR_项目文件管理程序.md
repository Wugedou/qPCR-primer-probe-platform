# qPCR 引物探针设计仿真平台 — 项目文件管理程序

> 定义本项目文件的存放、使用、更新、新建、删除规则

## 一、适用范围

本项目所有工作文件，包括代码、验证脚本、调研文献、流水账、报告等。

## 二、文件分级

| 层 | 文件 | 用途 | 修订规则 |
|:---|:---|:---|:---|
| Program File | 本文件 | 定义运作规则 | 修订另存新版, 原版标 SUPERSEDED |
| SOP | SOP-<编号>_<标题>_vX.Y_YYYYMMDD.md | 标准工作流程 | 修订另存新版, 原版标 SUPERSEDED |
| 仓库清单 | REPO-INVENTORY_仓库清单.md | 全部文件 R-NNNN 索引 | 每次新建/更新/删除文件后同步更新 |
| 流水账 | YYYY-MM-DD_简述.md | 按时间记录关键事件 | 追加, 不修改已记录事件 |
| PLAN | PLAN-<编号> | 任务计划 §1-§8 | 可修订, 记入流水账 |
| REPORT | REPORT_PLAN-<编号>_* | 每步交付物 | 完成后不再修改 |

## 三、存放位置

```
qPCR_引物探针设计仿真平台/
├── _shared/
│   ├── INDEX.md                   ← 本项目文件索引
│   ├── PROGRAM-QPCR_项目文件管理程序.md  ← 本文件
│   ├── SOP/                       ← 项目专属 SOP
│   ├── 流水账/                    ← YYYY-MM-DD_简述.md
│   ├── 调研文献/                  ← 文献摘要、真实数据源
│   └── 验证报告/                  ← REPORT_*.md
├── core/                          ← 核心代码(非文档, 不登记 R-NNNN)
├── validation/                    ← 验证脚本(非文档, 不登记 R-NNNN)
└── data/                          ← 数据文件(非文档, 不登记 R-NNNN)
```

## 四、工作流程

任何专项工作:
1. 查阅本程序 → 确认规则
2. 查阅对应 SOP(若有) → 按步骤执行
3. 执行过程中同步更新仓库清单 + 流水账
4. 产出 REPORT
5. 需修改 SOP/Program File → 另存新版本, 原版标 SUPERSEDED, 变更记入流水账

## 五、仓库清单登记规则

- 每个新文档分配 R-NNNN 编号
- 代码文件(.py)、数据文件(.json/.csv) 不登记
- 仅 .md 文档登记
- 格式: `| R-NNNN | 文件名 | 路径 | 创建日期 | 说明 |`

## 六、灵活性原则

SOP 是指引不是枷锁:
- ✅ 可改: 发现 SOP 不适用/有错误/遗漏时, 执行者可灵活处理
- ✅ 但要留痕: (1) 另存新版本, 原版标 SUPERSEDED; (2) 流水账记录变更
- ❌ 不能做的是: 悄悄改不留痕
